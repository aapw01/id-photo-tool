## 2. 设计哲学

### 2.1 三个关键词

```text
精确 · Precise        证件照核心是"严格符合规格"，每个像素都不能将就
清晰 · Clear          用户做完每一步都要明白"我刚才做了什么 / 下一步是什么"
克制 · Restrained     工具的颜值在留白与节奏，不在装饰
```

### 2.2 反例（我们绝不做）

- 渐变 banner + 居中大标题 + 大箭头按钮（国内工具常见 1999 风）
- 五颜六色的 CTA 按钮拼图
- 卡通插画 / emoji 满天飞
- 中文字体用宋体或楷体作为装饰
- "立即免费试用！" 这种话术
- 营销弹窗、强制注册、邮件订阅引导
- **任何位置使用 emoji 表情**（包括功能图标、状态指示、装饰、国旗都不用 emoji；用 `lucide-react` 与 `flag-icons` 这两个图标库统一替代）

### 2.3 参考品牌（风格基线）

| 品牌   | 学什么                           |
| ------ | -------------------------------- |
| Linear | 留白节奏、信息密度、暗黑模式克制 |
| Vercel | 字体规模、几何感、网格秩序       |
| Stripe | 表单细节、错误反馈、空状态文案   |
| Figma  | 工作台分区、面板交互、键盘快捷   |
| Notion | 文案温度、引导友好               |

## 3. 品牌视觉

### 3.1 Logo

> 待设计。初版可用 wordmark：`Pixfit` 全小写、Inter 字体、`@font-weight: 700`、字距 `-0.02em`、主色 emerald-500。

设计方向：

- 几何感（与"精确"呼应）
- 单色单形（emerald 一种颜色就够）
- 可压缩成 favicon（16×16 可识别）

可探索方向（备选，不做约束）：

- 字母 `i` 上的点变成"裁剪框"小方块
- `fi` 连字处藏一个像素方格

### 3.2 配色系统

#### 3.2.1 品牌色

| 角色      | Token            | Hex       | 用途                         |
| --------- | ---------------- | --------- | ---------------------------- |
| 主色      | `--primary`      | `#10B981` | 主行动按钮、链接、品牌识别   |
| 主色 - 深 | `--primary-dk`   | `#059669` | hover、active                |
| 主色 - 浅 | `--primary-soft` | `#D1FAE5` | 选中态背景、focus ring 衬底  |
| 强调色    | `--accent`       | `#F59E0B` | 重要提示、合规警告（非错误） |

#### 3.2.2 中性色

| 角色     | Token                | Hex                | 用途             |
| -------- | -------------------- | ------------------ | ---------------- |
| 页面背景 | `--bg`               | `#FAFAF9`          | body             |
| 卡片表面 | `--surface`          | `#FFFFFF`          | card / panel     |
| 浮层表面 | `--surface-elevated` | `#FFFFFF` + shadow | dialog / popover |
| 主文本   | `--text`             | `#1C1917`          | 标题、正文       |
| 次要文本 | `--text-mute`        | `#57534E`          | 说明、辅助       |
| 弱文本   | `--text-weak`        | `#A8A29E`          | 占位、disabled   |
| 边框     | `--border`           | `#E7E5E4`          | 卡片、输入框     |
| 分隔线   | `--divider`          | `#F5F5F4`          | 列表分隔         |

#### 3.2.3 语义色

| 角色 | Token       | Hex       | 用途                 |
| ---- | ----------- | --------- | -------------------- |
| 成功 | `--success` | `#10B981` | 与品牌色同源         |
| 警告 | `--warning` | `#F59E0B` | 合规警告、慎重操作   |
| 错误 | `--danger`  | `#EF4444` | 表单错误、失败 toast |
| 信息 | `--info`    | `#0EA5E9` | 提示性信息           |

#### 3.2.4 暗黑模式（V1.1 可选）

V1 先做明亮模式。暗黑模式预留 CSS variable 切换机制，不实际实现。

### 3.3 字体

| 用途       | 字体                                    | Weight                | 备注                |
| ---------- | --------------------------------------- | --------------------- | ------------------- |
| 英文       | `Inter Variable`                        | 400 / 500 / 600 / 700 | 自托管 `.woff2`     |
| 中文（简） | `PingFang SC` → `Noto Sans SC` fallback | 400 / 500 / 600 / 700 | 系统优先            |
| 中文（繁） | `PingFang TC` → `Noto Sans TC` fallback | 同上                  | 系统优先            |
| 代码       | `JetBrains Mono`                        | 400 / 500             | 仅在 KaTeX / 代码块 |

字号比例（rem 基于 16px）：

| 名称        | 字号            | 行高 | 用途         |
| ----------- | --------------- | ---- | ------------ |
| `display-1` | 3.5rem (56px)   | 1.1  | Hero 大标题  |
| `display-2` | 2.5rem (40px)   | 1.15 | 次要 hero    |
| `h1`        | 2rem (32px)     | 1.2  | 页面 H1      |
| `h2`        | 1.5rem (24px)   | 1.3  | section 标题 |
| `h3`        | 1.25rem (20px)  | 1.4  | 卡片标题     |
| `body-lg`   | 1.125rem (18px) | 1.6  | 大正文       |
| `body`      | 1rem (16px)     | 1.6  | 默认正文     |
| `body-sm`   | 0.875rem (14px) | 1.5  | 辅助文字     |
| `caption`   | 0.75rem (12px)  | 1.4  | 标签、说明   |

字重默认 500（中性），强调用 600，标题 700。**不用 400 以下**（太细，中文渲染差）。

### 3.4 间距系统

8px 基础栅格，常用倍数：

```text
xs   4px        紧凑控件内部间距
sm   8px        小图标与文字
md   12px       表单 label 与 input
lg   16px       默认 padding
xl   24px       卡片内部
2xl  32px       section 之间
3xl  48px       页面块之间
4xl  64px       Hero 上下
5xl  96px       大型留白
```

### 3.5 圆角

| Token         | 值     | 用途                   |
| ------------- | ------ | ---------------------- |
| `radius-sm`   | 6px    | 标签、小按钮           |
| `radius-md`   | 8px    | 输入框、下拉、tooltip  |
| `radius-lg`   | 12px   | **按钮（默认）、卡片** |
| `radius-xl`   | 16px   | 画布容器、大卡片       |
| `radius-2xl`  | 24px   | 弹窗、特殊容器         |
| `radius-full` | 9999px | 头像、徽章、进度环     |

### 3.6 阴影

柔和的双层阴影，**不要硬阴影、不要色彩阴影**：

| Token         | 值                                | 用途                       |
| ------------- | --------------------------------- | -------------------------- |
| `shadow-sm`   | `0 1px 2px rgba(0,0,0,0.04)`      | 卡片、输入 hover           |
| `shadow-md`   | `0 4px 12px rgba(0,0,0,0.06)`     | 浮层、popover              |
| `shadow-lg`   | `0 12px 32px rgba(0,0,0,0.08)`    | dialog、command palette    |
| `shadow-glow` | `0 0 0 4px rgba(16,185,129,0.15)` | focus ring（emerald 衬底） |

### 3.7 动效

- 默认过渡：`200ms cubic-bezier(0.4, 0, 0.2, 1)`
- 慢过渡（大尺寸/打开关闭）：`300ms`
- 微动效（按下/反馈）：`100ms`
- **永远不要旋转 spinner**：用进度环（确定的）或脉冲（不确定的）
- 页面切换：100ms fade，不做 slide
- 列表入场：错开 30ms 的 stagger，最多 8 项

### 4.3 国旗

**全站不使用 emoji**（包括国旗 emoji）。原因：emoji 渲染依赖系统字体，跨平台不一致；与 Lucide 线性图标风格冲突；与 emerald 单色品牌调性冲突。

使用 `flag-icons` 库（SVG 国旗集合，按需 tree-shake）：

```ts
import 'flag-icons/css/flag-icons.min.css'

// 圆形国旗组件（自定义封装）
function RegionFlag({ code, size = 20 }: { code: string; size?: number }) {
  return (
    <span
      className={`fi fi-${code.toLowerCase()} rounded-full`}
      style={{ width: size, height: size, display: 'inline-block' }}
      aria-label={`flag of ${code}`}
      role="img"
    />
  )
}
```

国家代码映射（ISO 3166-1 alpha-2 + 自定义）：

```ts
type RegionCode =
  | 'cn'
  | 'us'
  | 'gb'
  | 'jp'
  | 'kr'
  | 'sg'
  | 'my'
  | 'th'
  | 'vn'
  | 'ru'
  | 'au'
  | 'nz'
  | 'ca'
  | 'hk'
  | 'mo'
  | 'tw'
  | 'eu' // 申根用欧盟旗（flag-icons 支持）
```

样式：

- 默认 20×20 圆形（在列表、chip 中）
- 大尺寸 32×32（在尺寸专题页头部）
- 描边：1px `var(--border)`，避免白色国旗在白底融化

- **不放视频、不放轮播图、不放第三方推荐 logo**

## 6. 关键交互模式

### 6.1 上传区

**状态**：

- 默认：虚线边框 emerald-300，上传图标 + 文字 + 格式说明
- Hover（鼠标进入）：边框变实线 emerald-500，背景 emerald-50
- DragOver（拖拽进入）：边框 emerald-600，背景 emerald-100，缩放 1.02，shadow-glow
- 上传中：进度环 + "解析中..."
- 错误：边框 danger，错误文案 + 重试按钮

### 6.2 进度提示（抠图过程）

```text
不要做：
  spinner + "加载中..."

要做：
  ┌──────────────────────────────┐
  │   [Progress Ring 65%]        │  ← SVG 进度环（自定义组件）
  │   下载抠图模型               │  ← 当前阶段
  │   3.9 / 6.0 MB · 国内 CDN    │  ← 详情，让用户知道在做什么
  └──────────────────────────────┘
```

三阶段：

1. `下载模型 (首次)` 或 跳过 `(已缓存)`
2. `初始化推理引擎`
3. `处理图像 (WebGPU)`

### 6.3 撤销 / 重做

- 键盘：`Cmd/Ctrl + Z` / `Cmd/Ctrl + Shift + Z`
- 历史最多 50 步
- 大状态切换（如切换不同 PhotoSpec）不进历史，只记录每个 spec 内部的微调

### 6.4 撤销栈策略

```text
进历史栈的操作：
  ✓ 切换底色
  ✓ 调整裁剪框位置/大小
  ✓ 旋转
  ✓ 选定不同 PhotoSpec（作为锚点）

不进历史栈：
  ✗ 切换 tab（只是 UI 状态）
  ✗ 滚动缩放
  ✗ 修改规格库
```

### 6.5 Toast 通知

```text
位置：右下角，距底 24px、距右 24px
宽度：320px
持续：3s（成功）/ 5s（警告）/ 7s（错误，可手动关闭）
堆叠：最多 3 个，最新在底部，旧的向上推

颜色与图标（全部 Lucide）：
  成功  → 主色描边 + Lucide `CheckCircle2`     (emerald)
  警告  → amber 描边 + Lucide `AlertTriangle` (amber)
  错误  → danger 描边 + Lucide `XCircle`       (danger)
  信息  → info 描边 + Lucide `Info`            (info)
```

### 6.6 表单 & 错误反馈

- 失败的输入：实时（onBlur）显示红色边框 + 下方红色文案
- 文案规范：动词 + 名词 + 期望值（不要 "Invalid input"）
  - ✓ `请输入 1–500 之间的数字`
  - ✗ `输入无效`

### 6.7 空状态

每个空状态都要：

- 一个图标
- 一句标题（说当前是什么状态）
- 一句副标题（说为什么）
- 一个行动（怎么离开这个状态）

```text
┌────────────────────────┐
│   [Inbox icon 48px]    │  ← Lucide Inbox / ImagePlus
│                        │
│   还没有上传照片         │
│   开始你的第一张证件照    │
│                        │
│   [上传照片]            │
└────────────────────────┘
```

### 8.1 语气

- **专业但不冷漠**：像懂行的朋友帮你办手续，不像客服机器人

## 10. 可访问性细则

- 所有交互元素 `:focus-visible` 必须有 `shadow-glow`
- 关键按钮 `aria-label` 写清楚（图标按钮必填）
- 表单错误 `aria-describedby` 关联到错误文本
- 色板按钮：除颜色还要有名称（emerald `aria-label="emerald 500"`）
- 弹窗：`role="dialog"`、`aria-modal="true"`、初始焦点在第一个可交互元素
- Toast：`role="status"`（成功/信息）或 `role="alert"`（错误）
- 跳过链接：`<a href="#main">跳到主内容</a>` 仅键盘焦点时显示
